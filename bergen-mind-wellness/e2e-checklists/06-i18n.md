# Priority 6: Internationalization (i18n)

> **43 Bilingual Support Tests**
> English/Spanish language switching, translation accuracy, and locale-specific formatting

[← Back to Main Checklist](../E2E-MANUAL-TESTING-CHECKLIST.md)

---

## Test Suite Overview

This checklist validates comprehensive bilingual support:
- **Language Switcher** (6 tests)
- **Content Translation** (12 tests)
- **Date & Time Formatting** (8 tests)
- **Form Validation Messages** (7 tests)
- **URL Structure** (5 tests)
- **Persistent Language Preference** (5 tests)

---

## Prerequisites

### Environment Setup
- [ ] Dev server running at `http://localhost:3000`
- [ ] Both English and Spanish translations loaded
- [ ] Browser locale can be changed (for testing detection)

### Supported Languages
- **English** (`en`): Default language
- **Spanish** (`es`): Spanish (Latin America or Spain - note variation in testing)

### URL Structure
- English pages: `http://localhost:3000/en/...`
- Spanish pages: `http://localhost:3000/es/...`

### Critical Issues Found

1. **Calendar Not Localized** (I18N-20, I18N-21, I18N-22)
   - Month names stay in English ("November" instead of "Noviembre")
   - Day names stay in English ("Mon, Tue, Wed" instead of "Lun, Mar, Mié")
   - Full date display not translated ("Monday, November 24, 2025")
   - **Fix needed**: Pass locale prop to calendar component

2. **Form Validation Messages Not Translated** (I18N-28, I18N-30, I18N-32)

## Common Issues & Fixes

### Issue: Some text not translated
**Cause**: Hardcoded strings in components
**Fix**: Use i18n library (next-intl, react-i18next)
```javascript
// BAD
<h1>Book an Appointment</h1>

// GOOD
<h1>{t('booking.title')}</h1>
```

### Issue: Date format not changing with language
**Cause**: Using hardcoded format
**Fix**: Use locale-aware date formatting
```javascript
// Use Intl.DateTimeFormat
new Intl.DateTimeFormat('es-ES').format(date); // DD/MM/YYYY
new Intl.DateTimeFormat('en-US').format(date); // MM/DD/YYYY
```

### Issue: Translation files missing keys
**Cause**: Incomplete translation files
**Fix**: Ensure parity between en.json and es.json
```json
// en.json
{ "booking.title": "Book an Appointment" }

// es.json
{ "booking.title": "Reservar una Cita" }
```

### Issue: Language switcher doesn't work
**Cause**: Missing routing configuration
**Fix**: Configure Next.js i18n routing in next.config.js
```javascript
module.exports = {
  i18n: {
    locales: ['en', 'es'],
    defaultLocale: 'en',
  },
};
```
